# 维护说明（长期更新规范）

## 一、每日更新流程（固定）

1. 推荐执行脚本：`./new_problem.sh <leetcode_no> <title_short> [difficulty] [category]`
2. 检查生成目录和文件是否正确。
3. 完整填写 `README.md`（题目、示例、边界、思路、白话讲解）。
4. 完整填写 `SolutionLC<题号>.java`（详细注释）。
5. 完整填写 `SolutionLC<题号>.py`（详细注释）。
6. 更新 `INDEX.md`（新增一行记录）。
7. 题目若有图，执行 `python3 sync_leetcode_images.py <README路径>` 同步题面示意图。

## 二、目录命名规范

- 年目录：`YYYY`，如 `2026`
- 月目录：`MM`，如 `03`
- 题目目录：`YYYY-MM-DD-leetcode-编号-题目短名`
  - 示例：`2026-03-26-leetcode-3-无重复字符最长子串`
  - 题目短名可中文，但建议简短，避免过长路径

## 三、内容质量最低要求

- 题目描述清晰，不省略关键条件。
- 至少 2 个输入输出示例。
- 明确边界条件（空数组、最小值、最大值、重复值等）。
- 思路讲解必须包含：
  - 为什么想到这个解法
  - 时间复杂度与空间复杂度
  - 为什么不会漏解或误解
- 若存在多层次解法，必须写全：一般解法、经典解法、最优解法。
- README 中至少给出一般解法与最优解法的复杂度对比。
- 代码注释要解释决策，不只翻译代码行为。

## 四、一致性要求

- 每题目录内文件名固定不变：
  - `README.md`
  - `SolutionLC<题号>.java`
  - `SolutionLC<题号>.py`
- 代码顺序固定：Java 在前，Python 在后。
- 难度只使用：`Easy` / `Medium` / `Hard`。
- 题型尽量使用 `INDEX.md` 的统一枚举。

## 五、长期维护建议

- 每月底回顾当月题目，检查是否有缺注释、缺边界、缺索引的问题。
- 每季度可新增一个 `monthly-summary` 文档复盘高频题型和薄弱点。
- 不要重构历史目录命名，避免链接和检索失效。

## 六、脚本使用注意事项

- 脚本路径：`new_problem.sh`
- 首次使用若无执行权限，请运行：`chmod +x new_problem.sh`
- 传入参数中 `title_short` 可中文，脚本会自动将空格替换为 `-`。
- 若当天已存在同名目录，脚本会拒绝覆盖，避免误删历史内容。
- 图片同步脚本：`sync_leetcode_images.py`（会把题面图下载到题目目录的 `images/`）。
