# LeetCode 53 - 最大子数组和

## 基本信息

- 日期：2026-04-03
- 难度：Medium
- 题型：动态规划
- 题目链接：[https://leetcode.com/problems/maximum-subarray/](https://leetcode.com/problems/maximum-subarray/)

## 题目描述

给定整数数组 `nums`，找出一个具有最大和的 **连续** 子数组（子数组最少包含一个元素），返回其最大和。

## 输入输出示例

### 示例 1

- 输入：`nums = [-2,1,-3,4,-1,2,1,-5,4]`
- 输出：`6`
- 解释：连续子数组 `[4,-1,2,1]` 的和最大，为 `6`。

### 示例 2

- 输入：`nums = [1]`
- 输出：`1`

### 示例 3

- 输入：`nums = [5,4,-1,7,8]`
- 输出：`23`

## 边界条件

- 数组长度至少为 1
- 元素可为负数，最大和子数组可能只含一个元素
- 若全为负，答案为其中最大（绝对值最小的负）的那个元素

## 多解法

### 一般解法

- 核心思路：枚举所有区间 `[i, j]`，求区间和取最大。可用三重循环或二重循环 + 前缀和。
- 时间复杂度：`O(n^2)`（前缀和优化后）或 `O(n^3)`（朴素三重）
- 空间复杂度：`O(1)` 或 `O(n)`（若存前缀和数组）

### 经典解法

- 核心思路：**分治**。最大子段和要么全在左半、全在右半，或跨越中点；递归求解。时间 `O(n log n)`。
- 时间复杂度：`O(n log n)`
- 空间复杂度：`O(log n)` 递归栈

### 最优解法

- 核心思路：**Kadane（动态规划）**。维护「以当前元素结尾的最大子数组和」：`cur = max(nums[i], cur + nums[i])`，答案为所有 `cur` 的最大值。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 解题思路（以 Kadane 为主）

### 思路概述

若「以 `i-1` 结尾的子数组和」已经小于 0，对 `nums[i]` 来说不如重新开始一段，只包含 `nums[i]`。  
因此状态转移：`cur = max(nums[i], cur + nums[i])`，全局 `ans = max(ans, cur)`。

### 关键步骤

1. 初始化 `ans = cur = nums[0]`
2. 从 `i = 1` 遍历到末尾，更新 `cur` 与 `ans`
3. 返回 `ans`

## 通俗白话讲解

从左往右走，累加当前这段的和。  
如果累加已经变成「包袱」（和为负），就扔掉包袱从当前数重新开张。  
一路上记录「开张以来最好成绩」的全局最大值。

## 复杂度分析（对比）

- 一般解法：时间 `O(n^2)`，空间 `O(1)`（或 `O(n)` 前缀数组）
- 最优解法：时间 `O(n)`，空间 `O(1)`
