# LeetCode 15 - 三数之和

## 基本信息

- 日期：2026-04-05
- 难度：Medium
- 题型：双指针
- 题目链接：[https://leetcode.com/problems/3sum/](https://leetcode.com/problems/3sum/)

## 题目描述

给定整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k`、`j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有和为 `0` 且不重复的**三元组**。

**注意：** 答案中不可以包含重复的三元组。

## 输入输出示例

### 示例 1

- 输入：`nums = [-1,0,1,2,-1,-4]`
- 输出：`[[-1,-1,2],[-1,0,1]]`
- 解释：`nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0`，且三元组不重复。

### 示例 2

- 输入：`nums = [0,1,1]`
- 输出：`[]`

### 示例 3

- 输入：`nums = [0,0,0]`
- 输出：`[[0,0,0]]`

## 边界条件

- 数组长度可能小于 3，此时无解
- 需要去重：排序后跳过相同 `nums[i]`，找到一组解后收缩 `left`/`right` 时也要跳过重复值

## 多解法

### 一般解法

- 核心思路：三重循环枚举 `i < j < k`，用 `HashSet` 存已得三元组（排序后字符串）去重。
- 时间复杂度：`O(n^3)`
- 空间复杂度：`O(结果数量)` 或额外 `O(n^3)` 级别（不推荐）

### 经典解法

- 核心思路：先**排序**，固定第一个下标 `i`，在右侧区间用**双指针** `left`、`right` 找两数之和为 `-nums[i]`；移动时利用有序性跳过重复元素。
- 时间复杂度：`O(n^2)`（排序 `O(n log n)`）
- 空间复杂度：`O(1)` 额外（不计结果与排序栈）

### 最优解法

- 核心思路：与「排序 + 双指针」相同，已是本题主流最优时间复杂度。

## 解题思路（以排序 + 双指针为主）

### 思路概述

排序后，枚举 `i`，问题变为在 `i+1 .. n-1` 上找两数之和等于 `-nums[i]`。  
双指针：和小于目标则 `left++`，大于则 `right--`，相等则记录并两侧同时收缩且跳过重复值。

### 关键步骤

1. `Arrays.sort(nums)`
2. `i` 从 `0` 到 `n-3`，若 `nums[i] == nums[i-1]` 则跳过
3. `left = i+1`，`right = n-1`，按和与 `0` 比较移动指针
4. 命中时加入三元组，`left`/`right` 各自跳过连续相同值后再移动

## 通俗白话讲解

先排队（排序），盯住最左边一个人当「队长」，再在右边队伍里用两头夹的办法找两个人，看三人身高（数值）加起来是不是刚好为 0。  
队长和左右两人如果和前一轮一模一样，就跳过，避免重复报同一组名单。

## 复杂度分析（对比）

- 一般解法：时间 `O(n^3)`，空间视去重实现而定
- 最优解法：时间 `O(n^2)`，额外空间 `O(1)`
