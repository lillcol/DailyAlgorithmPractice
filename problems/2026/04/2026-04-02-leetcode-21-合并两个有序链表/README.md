# LeetCode 21 - 合并两个有序链表

## 基本信息

- 日期：2026-04-02
- 难度：Easy
- 题型：链表
- 题目链接：[https://leetcode.com/problems/merge-two-sorted-lists/](https://leetcode.com/problems/merge-two-sorted-lists/)

## 题目描述

将两个升序链表合并为一个新的 **升序** 链表并返回。新链表由原来两个链表的节点拼接而成。

## 输入输出示例

### 示例 1

- 输入：`list1 = [1,2,4]`，`list2 = [1,3,4]`
- 输出：`[1,1,2,3,4,4]`

### 示例 2

- 输入：`list1 = []`，`list2 = []`
- 输出：`[]`

### 示例 3

- 输入：`list1 = []`，`list2 = [0]`
- 输出：`[0]`

## 边界条件

- 其中一个链表可能为空
- 两个都为空时返回空链表
- 节点值可能相等，合并后保持非降序（通常取 `<=` 时先接 `l1` 以稳定）

## 多解法

### 一般解法

- 核心思路：遍历两条链表把值放进数组（或列表），排序后再依次建链。
- 时间复杂度：`O((m+n) log(m+n))`（排序主导）
- 空间复杂度：`O(m+n)`（存所有值）

### 经典解法

- 核心思路：**哑结点 + 双指针**。每次比较 `l1`、`l2` 表头，较小者接到结果链尾部并后移；有一链走完后，把另一链剩余整体接上。
- 时间复杂度：`O(m+n)`
- 空间复杂度：`O(1)`（仅若干指针，不计返回链表本身）

### 最优解法

- 核心思路：与经典迭代相同，已是本题最优时间；也可用递归写法，思路一致但隐式栈为 `O(m+n)`，通常面试更推荐迭代。

## 解题思路（以经典迭代为主）

### 思路概述

维护结果链表的尾指针 `cur`，初始为哑结点 `dummy`。  
每一步在 `l1`、`l2` 当前头结点中选较小（相等时任选一侧，代码里常先接 `l1`）接到 `cur.next`，并移动对应链表指针。  
当一条链耗尽，`cur.next` 指向另一条链剩余部分即可。

### 关键步骤

1. `dummy`、`cur` 初始化，`cur` 从 `dummy` 开始
2. `while (l1 != null && l2 != null)` 比较头结点值，接小的一侧，`cur` 后移
3. 循环结束后 `cur.next = l1 != null ? l1 : l2`
4. 返回 `dummy.next`

## 通俗白话讲解

像合并两条已排好队的队伍：每次从两个队首里挑出更靠前的那个人排到新队伍里。  
一队没人了，剩下那队直接接到新队伍末尾就行。

## 复杂度分析（对比）

- 一般解法：时间 `O((m+n) log(m+n))`，空间 `O(m+n)`
- 经典解法：时间 `O(m+n)`，额外空间 `O(1)`
