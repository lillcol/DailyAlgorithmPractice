# LeetCode 146 - LRU 缓存

## 基本信息

- 日期：2026-04-12
- 难度：Medium
- 题型：哈希表
- 题目链接：[https://leetcode.com/problems/lru-cache/](https://leetcode.com/problems/lru-cache/)

## 题目描述

请你设计并实现一个满足 **LRU（最近最少使用）** 缓存约束的数据结构。

实现 `LRUCache` 类：

- `LRUCache(int capacity)` 以正整数作为容量 `capacity` 初始化 LRU 缓存
- `int get(int key)` 如果关键字 `key` 存在于缓存中，则返回关键字的值，否则返回 `-1`
- `void put(int key, int value)` 如果关键字 `key` 已经存在，则变更其数据值 `value`；如果不存在，则向缓存中插入该组 `key-value`。如果插入操作导致关键字数量超过 `capacity`，则 **逐出** 最久未使用的关键字

函数 `get` 和 `put` 必须以 **平均时间复杂度为 O(1)** 完成。

## 输入输出示例

题目为交互式调用，常见顺序（`capacity = 2`）：

- `put(1, 1)`，`put(2, 2)`，`get(1)` 返回 `1`（键 `1` 仍为最近使用）
- `put(3, 3)` 逐出键 `2`，`get(2)` 返回 `-1`
- `put(4, 4)` 逐出键 `1`，`get(1)` 返回 `-1`，`get(3)` 返回 `3`，`get(4)` 返回 `4`

## 边界条件

- `capacity >= 1`
- `get` 不存在的键返回 `-1`
- `put` 更新已存在键时 **不增加** 当前条目数，但需将该键标为「最近使用」
- 容量满时再 `put` 新键，必须先淘汰 **最久未使用** 的条目

## 多解法

### 一般解法

- 核心思路：用数组或链表存 `(key, value)`，每次 `get`/`put` 线性查找；用额外规则标记 LRU 顺序。单次操作 `O(n)`，不满足题意。
- 时间复杂度：`get`、`put` 均为 `O(n)`
- 空间复杂度：`O(capacity)`

### 经典解法

- 核心思路：**哈希表 + 双向链表**。哈希表 `key -> 链表结点`，实现 O(1) 定位；双向链表按「使用时间」维护顺序（例如靠近尾端为最近使用，靠近头端为最久未使用）。`get` 命中则把结点移到「最近使用」端；`put` 若超容量则删掉「最久未使用」端结点并从表中移除映射。
- 时间复杂度：`get`、`put` 平均 `O(1)`
- 空间复杂度：`O(capacity)`

### 最优解法

- 与经典一致。Python 也可用 `OrderedDict` 的 `move_to_end` 模拟顺序（面试/题解常要求能手写双向链表，本仓库代码为 **手写双向链表 + 哈希表**）。

## 解题思路

### 思路概述

要同时做到「按 key 快速查找」和「快速删除最久未使用 / 调整最近使用顺序」，单用哈希表或单用链表都不够；二者结合：表负责定位，链表负责时间顺序。

### 关键步骤

1. 使用哑头、哑尾结点，避免边界上的空指针判断。
2. `get`：key 不在表则 `-1`；在则读出值，把该结点从链表摘下再接到「最近使用」侧。
3. `put`：key 已存在则改 `value` 并移到最近使用侧；不存在则新建结点；若已满，先删掉 LRU 端结点并删除表中对应 key，再插入新结点。

## 通俗白话讲解

缓存像一排格子，常用的放顺手边，最久没碰的在最远处。  
满了要新进一个时，只能扔掉「最久没人动」的那一个。  
哈希表让你瞬间找到某个 key 在哪一格；双向链表让你随时把某一格挪到「刚用过」的位置，并快速扔掉最远那一格。

## 复杂度分析（对比）

- 朴素线性结构：每次 `O(n)`，不符合要求。
- 哈希表 + 双向链表：`get`、`put` 均 `O(1)`，额外空间 `O(capacity)`。
