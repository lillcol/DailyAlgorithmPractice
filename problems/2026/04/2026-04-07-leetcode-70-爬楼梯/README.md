# LeetCode 70 - 爬楼梯

## 基本信息

- 日期：2026-04-07
- 难度：Easy
- 题型：动态规划
- 题目链接：[https://leetcode.com/problems/climbing-stairs/](https://leetcode.com/problems/climbing-stairs/)

## 题目描述

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 `1` 或 `2` 个台阶。你有多少种**不同**的方法可以爬到楼顶？

## 输入输出示例

### 示例 1

- 输入：`n = 2`
- 输出：`2`
- 解释：两种方法：`1 + 1`、`2`

### 示例 2

- 输入：`n = 3`
- 输出：`3`
- 解释：三种方法：`1 + 1 + 1`、`1 + 2`、`2 + 1`

## 边界条件

- `n` 为正整数，通常 `1 <= n <= 45`（不同版本约束略有不同）
- `n = 1` 时答案为 `1`

## 多解法

### 一般解法

- 核心思路：**暴力递归** `f(n) = f(n-1) + f(n-2)`，无记忆化，大量重复子问题。
- 时间复杂度：指数级，约 `O(2^n)`
- 空间复杂度：递归栈 `O(n)`

### 经典解法

- 核心思路：**线性 DP**。`dp[i]` 表示到达第 `i` 阶的方案数，`dp[i] = dp[i-1] + dp[i-2]`，初值 `dp[1]=1`，`dp[2]=2`（或统一从 `dp[0]=1` 推导）。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`（DP 数组）

### 最优解法

- 核心思路：注意到状态只依赖前两项，用**两个变量滚动**更新，等价于斐波那契数列移位。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 解题思路（以滚动变量为主）

### 思路概述

到达第 `n` 阶的最后一步，要么从 `n-1` 迈 1 步上来，要么从 `n-2` 迈 2 步上来，两类方案互不重复且覆盖全部情况，故 `ways(n) = ways(n-1) + ways(n-2)`。

### 关键步骤

1. 初始化 `prev2 = 1`（可理解为 `ways(0)` 或起点方案数）、`prev1 = 1`（`ways(1)`）
2. 从 `2` 循环到 `n`，每次 `cur = prev1 + prev2`，再滚动 `prev2, prev1 = prev1, cur`
3. 返回 `prev1`（循环结束后表示 `ways(n)`）

## 通俗白话讲解

最后一步只有两种来历：差一阶上来，或差两阶上来。  
把「到前一阶的方法数」和「到前两阶的方法数」加起来，就是到当前阶的总方法数。

## 复杂度分析（对比）

- 一般递归：时间指数级，不推荐
- 滚动 DP：时间 `O(n)`，空间 `O(1)`
