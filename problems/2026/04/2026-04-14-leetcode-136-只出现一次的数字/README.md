# LeetCode 136 - 只出现一次的数字

## 基本信息

- 日期：2026-04-14
- 难度：Easy
- 题型：数组
- 题目链接：[https://leetcode.com/problems/single-number/](https://leetcode.com/problems/single-number/)

## 题目描述

给你一个 **非空** 整数数组 `nums`，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

你必须设计并实现线性时间复杂度、且只使用常数额外空间的算法解决此问题。

## 输入输出示例

### 示例 1

- 输入：`nums = [2,2,1]`
- 输出：`1`

### 示例 2

- 输入：`nums = [4,1,2,1,2]`
- 输出：`4`

### 示例 3

- 输入：`nums = [1]`
- 输出：`1`

## 边界条件

- 数组非空，且长度为奇数（成对出现 + 单独一个）
- 除了一个数出现一次，其余都恰好出现两次
- 结果唯一

## 多解法

### 一般解法

- 核心思路：用哈希表统计每个数字出现次数，再遍历找出次数为 `1` 的键。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`（哈希表存频次）

### 经典解法

- 核心思路：**排序**后线性扫描，检查相邻元素是否成对；或排序后每隔两个取一次（实现细节较多）。
- 时间复杂度：`O(n log n)`（排序主导）
- 空间复杂度：`O(1)` 或 `O(log n)`（取决于排序实现）

### 最优解法

- 核心思路：**异或（XOR）**。性质：`a ^ a = 0`，`a ^ 0 = a`，且异或满足交换律与结合律。数组全体异或后，成对数字会互相抵消为 `0`，最终剩下只出现一次的那个数。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`（题解代码实现此版本）

## 解题思路

### 思路概述

把所有数做异或累加；两两相同的数异或得 `0`，不影响结果；最后剩下的就是落单的数。

### 关键步骤

1. 初始化 `ans = 0`。
2. 遍历 `nums`，执行 `ans ^= x`。
3. 返回 `ans`。

## 通俗白话讲解

你可以把这题想象成「消消乐」：  
桌上有很多数字卡片，每种数字（除了一个倒霉蛋）都恰好有两张一样的。

规则是：

1. 两张一样的卡片一碰就消失（对应 `a ^ a = 0`）。  
2. 消失不影响其它卡片继续配对（对应异或的结合律、交换律）。  

所以代码里我们每一步其实在做：  
从头到尾把数字用异或“叠在一起”，成对的都会抵消掉，最后剩下的就是那张只出现一次的卡片。

## 复杂度分析（对比）

- 哈希表：时间 `O(n)`，空间 `O(n)`
- 排序：时间 `O(n log n)`，空间视排序而定
- 异或：时间 `O(n)`，空间 `O(1)`
