# LeetCode 22 - 括号生成

## 基本信息

- 日期：2026-04-11
- 难度：Medium
- 题型：回溯
- 题目链接：[https://leetcode.com/problems/generate-parentheses/](https://leetcode.com/problems/generate-parentheses/)

## 题目描述

数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 **有效的** 括号组合。有效括号需满足：任意前缀中左括号 `(` 的数量 **不少于** 右括号 `)` 的数量。

## 输入输出示例

### 示例 1

- 输入：`n = 3`
- 输出：`["((()))","(()())","(())()","()(())","()()()"]`

### 示例 2

- 输入：`n = 1`
- 输出：`["()"]`

## 边界条件

- `n >= 1`（按题意常见约束；实现上 `n = 0` 时部分约定返回 `[""]`，本题以官方样例为准）
- 结果字符串长度恒为 `2n`
- 合法序列个数为第 `n` 个卡特兰数，随 `n` 增大结果集增长很快

## 多解法

### 一般解法

- 核心思路：枚举所有由 `n` 个 `(` 与 `n` 个 `)` 组成的排列（或先选左括号位置再填右括号），再逐串校验是否为合法括号串。
- 时间复杂度：约 `O(C(2n,n) * n)` 量级（组合数乘校验长度），远大于只构造合法串的方法。
- 空间复杂度：除答案外 `O(n)` 用于当前串；答案本身不计入复杂度上界时常单独说明。

### 经典解法

- 核心思路：**回溯 / DFS**。维护已放置的左括号数 `open`、右括号数 `close`：若 `open < n` 可放 `(`；若 `close < open` 可放 `)`（保证任意前缀左不少于右）。长度到 `2n` 时收集答案。
- 时间复杂度：与合法括号串个数成正比，约为卡特兰数级别；每个合法串长度 `O(n)`。
- 空间复杂度：递归栈 `O(n)`，路径缓冲区 `O(n)`。

### 最优解法

- 在「只输出所有合法串」的前提下，回溯已是主流最优思路之一；不存在低于「输出总量」的算法。题解代码采用回溯。

## 解题思路

### 思路概述

每步只追加 `(` 或 `)`，用计数器约束：**左括号最多 `n` 个**；**右括号不能超过左括号已用个数**，这样永远不会出现前缀里 `)` 更多的非法情况。

### 关键步骤

1. 从空串开始 DFS，`open`、`close` 初值为 `0`。
2. `open < n` 时分支追加 `(`，递归 `open + 1`。
3. `close < open` 时分支追加 `)`，递归 `close + 1`。
4. 当 `open == close == n`（或串长 `2n`）时把当前串加入结果。

## 通俗白话讲解

你要拼 `2n` 个字符，但括号必须「随时都合法」。  
规则可以记成两句：左括号不能超标（总共就 `n` 个）；右括号不能比左括号先用完——否则前面已经出现 `)` 比 `(` 多的断档。

## 复杂度分析（对比）

- 暴力枚举 + 校验：时间很大，不实用。
- 回溯：时间与合法结果数量同阶，额外栈与路径为 `O(n)`。
