# LeetCode 2 - 两数相加

## 基本信息

- 日期：2026-03-30
- 难度：Medium
- 题型：链表
- 题目链接：[https://leetcode.com/problems/add-two-numbers/](https://leetcode.com/problems/add-two-numbers/)

## 题目描述

给你两个 **非空** 的链表，表示两个非负整数。数字最高位在链表头部，每位数字按 **逆序** 存储（个位在链表头）。  
请你将两数相加，并以相同形式返回一条表示和的链表。

可以假设除了数字 `0` 本身以外，链表表示的数不含前导零。

## 输入输出示例

### 示例 1

- 输入：`l1 = [2,4,3]`，`l2 = [5,6,4]`（分别表示 342 与 465）
- 输出：`[7,0,8]`（表示 807）
- 解释：`342 + 465 = 807`

### 示例 2

- 输入：`l1 = [0]`，`l2 = [0]`
- 输出：`[0]`

### 示例 3

- 输入：`l1 = [9,9,9,9,9,9,9]`，`l2 = [9,9,9,9]`
- 输出：`[8,9,9,9,0,0,0,1]`

## 边界条件

- 两条链表长度可能不同
- 相加后位数可能比两条链表都多一位（最高位进位）
- 节点值为 `0`～`9` 的整数

## 多解法

### 一般解法

- 核心思路：遍历两条链表转成整数（或字符串拼接再转大数），相加后再拆成链表。思路直观，但链表很长时整数会溢出或需要大数库，工程上也不如直接模拟稳妥。
- 时间复杂度：`O(max(m, n))`（遍历长度）
- 空间复杂度：取决于是否用大数；若用普通 `long` 可能溢出，不适用一般约束

### 经典解法

- 核心思路：模拟竖式加法——从表头（个位）同步往后走，维护 `carry`，每一位 `(a + b + carry) % 10` 作为新节点，`carry` 进位。
- 时间复杂度：`O(max(m, n))`
- 空间复杂度：`O(1)` 额外空间（输出链表不计入则通常写 `O(1)`；若计输出则为 `O(max(m,n))`）

### 最优解法

- 核心思路：与「经典解法」相同，一次遍历 + 哑结点挂链，代码最干净。
- 时间复杂度：`O(max(m, n))`
- 空间复杂度：`O(1)` 额外指针空间（新建结果链表长度 `O(max(m,n))`）

## 解题思路（以最优解法为主）

### 思路概述

用哑结点 `dummy` 指向结果链表的头部，指针 `cur` 不断向后接新节点。  
每一轮把 `l1`、`l2` 当前位的值（若已走完则为 0）与进位相加，得到当前位数字与新的进位。

### 关键步骤

1. 初始化 `dummy`、`cur`、`carry = 0`
2. 当 `l1`、`l2` 未走完 **或** `carry > 0` 时循环
3. 取当前位之和，新建节点 `sum % 10`，`carry = sum / 10`，`l1`/`l2` 各自后移
4. 返回 `dummy.next`

## 通俗白话讲解

就像小学列竖式加法，只不过数字是「链表从个位排到头」。  
你每次只看当前列两个数字和上一轮有没有进位，写下这一位的结果，把进位留给下一轮。  
某条链表先走完就当后面全是 0，最后如果还剩进位，再补一个节点。

## 复杂度分析（对比）

- 一般解法：依赖转数字或大数，易溢出或实现笨重，不推荐作为标准答案
- 最优解法：时间 `O(max(m, n))`，额外指针空间 `O(1)`（结果链表本身 `O(max(m, n))`）
