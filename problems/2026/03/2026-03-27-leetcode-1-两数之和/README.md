# LeetCode 1 - 两数之和

## 基本信息

- 日期：2026-03-27
- 难度：Easy
- 题型：哈希表
- 题目链接：[https://leetcode.com/problems/two-sum/](https://leetcode.com/problems/two-sum/)

## 题目描述

给定一个整数数组 `nums` 和一个目标值 `target`，请你在该数组中找出和为目标值的两个整数，并返回它们的数组下标。  
你可以假设每种输入只会对应一个答案，且同一个元素不能使用两次。

## 输入输出示例

### 示例 1

- 输入：`nums = [2,7,11,15], target = 9`
- 输出：`[0,1]`
- 解释：因为 `nums[0] + nums[1] = 2 + 7 = 9`

### 示例 2

- 输入：`nums = [3,2,4], target = 6`
- 输出：`[1,2]`
- 解释：因为 `nums[1] + nums[2] = 2 + 4 = 6`

## 边界条件

- 数组长度最小可能为 2
- 可能出现重复元素
- 目标值可能为负数

## 多解法

### 一般解法

- 核心思路：双重循环枚举所有下标对 `(i, j)`，判断 `nums[i] + nums[j] == target`。
- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(1)`

### 经典解法

- 核心思路：先排序后用双指针夹逼查找和为 `target` 的数对（需要额外保存原下标）。
- 时间复杂度：`O(n log n)`（排序主导）
- 空间复杂度：`O(n)`（保存值与原下标映射）

### 最优解法

- 核心思路：单次遍历 + 哈希表，边遍历边查找 `target - nums[i]` 是否已出现。
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 解题思路

### 思路概述

使用哈希表记录“已经遍历过的数字及其下标”。  
每访问一个新数字，就检查“目标值减当前值”的配对数字是否已出现。

### 关键步骤

1. 初始化哈希表 `value -> index`
2. 遍历数组，计算 `need = target - nums[i]`
3. 如果 `need` 在哈希表中，返回 `[哈希表中的下标, 当前下标]`
4. 否则把当前值和下标放进哈希表继续遍历

## 通俗白话讲解

可以把哈希表想成一本“已经见过谁”的通讯录。  
每来一个新数字，你就问通讯录里有没有“能跟它凑成 target 的那个人”。  
如果有，立刻配对成功；如果没有，就把它自己登记进通讯录，等后面的人来配。

## 复杂度分析（对比）

- 一般解法：时间 `O(n^2)`，空间 `O(1)`
- 最优解法：时间 `O(n)`，空间 `O(n)`
